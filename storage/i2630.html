A Cheatsheet of High-Frequency Git Commands

My day-to-day git commands after deleting SourceTree from my machine

Tunvir Rahman
Feb 11 · 2 min read

1. Initialize a repository and connecting to Remote (eg. bitbucket):

git init

2. Create a sample repo in remote (this case bitbucket)

3. Copy the underlined text, which is <your remote URL>:

4. Connect remote:

git remote add origin <YourRemoteURL> //Now your repo is connected to a remote repository

For tracking changes and staging/un-staging files

git status // Show you your current branch and Changed files
git diff <ChangedFileName> // Show the changes in the filegit add <YourchangedFile> // Stage a particular file 
git reset HEAD <YourchangedFile> // Unstage a particular file
git reset --hard // reset all to head
git add . // Stage all changed files

For commit, log, and push changes

git commit -m "Message"
git push origin <YourBranchName> // if remote not exist otherwise simply git pushgit log // list all logs step by step

For branching-related stuff

git branch <BranchName> //create sample branch
git push origin <BranchName>// push the branch to remote
git push origin --delete <BranchName> //Delete branch from remotegit branch -r// show all Remote branch
git branch // show all Local branch
git branch -a // All local and remote branchgit checkout -b <BranchName> // checkout to local branch
git fetch && git checkout <RemotebranchName>//remote branch Checkoutgit branch -d <BranchName>//remove branch git merge <YourLocalBranch>
git merge origin <YourRemoteBranch>

For staging

git stash save <myStashName> //Stash all staged changes
git stash list //List all stashed
git stash drop stash@{ID} // id from listed stash
git stash apply stash@{ID} // Merge stashed changes to branch

Thanks for reading.
